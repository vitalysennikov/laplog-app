# GIT.md — правила работы с git в этом проекте

## Версионирование

- Версию обновлять **только по явному запросу** пользователя
- Файл версии: `app/build.gradle.kts` (поля `versionCode` и `versionName`)
  - Исправления → третья цифра (0.19.5 → 0.19.6), versionCode +1
  - Новый функционал → вторая цифра (0.18.x → 0.19.0), versionCode +1
- Версию обновлять в том же коммите, что и код (не отдельным коммитом)
- Сообщение коммита: `Версия X.Y.Z` + в теле — список **всех изменений с предыдущей версии**, сгруппированных по разделам «Новое» и «Исправления». Для сбора использовать `git log --oneline <коммит-предыдущей-версии>..HEAD`

## Правила коммитов

- НЕ добавлять `Co-Authored-By` в сообщения коммитов
- Команду `git commit` запускать отдельно, не объединять с другими командами через `&&`

## Запуск git-команд

- Git-команды запускать напрямую: рабочий каталог уже установлен в корень проекта
- НЕ использовать `cd "..." &&` перед командами
- НЕ использовать `git -C "..."`
- Все shell-команды запускать без `cd "..."` prefix
