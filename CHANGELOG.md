## [0.19.5] - 2026-05-21

### Исправлено
- **Сохранение набора акцентов при переключении имени**
  - При переключении на другое имя и обратно набор акцентов больше не сбрасывается к DEFAULT
  - Причина: `loadNameToggles()` записывал акценты имени в глобальный ключ SharedPreferences, перезаписывая явный выбор пользователя
  - Решение: загрузка акцентов имени больше не изменяет глобальный ключ; он обновляется только при явном выборе пользователя
- **Список бэкапов**
  - Файлы с нестандартным именем (например, `laplog_backup_fixed.json`) теперь отображаются в списке
  - Причина: исключение при разборе даты из имени файла молча пропускало запись
  - Решение: при ошибке разбора даты используется дата изменения файла

### Улучшено
- **Уведомления**
  - Формат статуса: `⏱ Название — запущен` (добавлено тире между названием и статусом)
  - Текст уведомления «Screen stays on» вынесен в строковые ресурсы и переведён на все языки (RU: «Экран держим активным», ZH: 保持屏幕唤醒)
  - Отображение текущего круга на главном экране обновляется 10 раз в секунду (вместо 1 раза) — устранено отставание на 1 секунду

## [0.9.7] - 2025-11-19

### Исправлено
- **Уведомление при запуске приложения (исправление окончательное)**
  - Полностью устранена проблема с появлением уведомления "Stopwatch paused" при запуске приложения
  - Причина: onCreate() вызывал startStateListener(), который создавал Flow collectors
  - Collectors вызывали updateNotification() при первом получении значений, создавая уведомление
  - Это происходило даже если сервис создавался только для получения ACTION_STOP
  - Решение: state listener теперь запускается только когда секундомер активен
- **Уведомление в режиме ALWAYS ON**
  - В режиме ALWAYS ON (секундомер остановлен, экран не выключается) показывается статическое уведомление "Screen stays on"
  - Уведомление не содержит кнопок управления, так как секундомер не активен
  - Исправлена проблема когда вместо этого показывалось уведомление "Stopwatch paused" с кнопками Start и Stop
- **Экран выключается на паузе в режиме ALWAYS с dimBrightness=OFF**
  - Исправлена проблема когда экран выключался после затухания яркости при паузе секундомера
  - В режиме ALWAYS + dimBrightness=OFF на паузе должно работать так же, как в состоянии СТОП: яркость затухает, но экран не выключается
  - Причина: ACTION_PAUSE освобождал все wake locks, включая screenDimWakeLock
  - Решение: в ACTION_PAUSE сохраняется screenDimWakeLock если useScreenDim=true (режим ALWAYS + dimBrightness=OFF)

### Технические детали
- **StopwatchService.kt**:
  - Убран вызов `startStateListener()` из метода `onCreate()`
  - Добавлены вызовы `startStateListener()` в:
    - ACTION_START: перед `startForeground()` - секундомер запущен
    - ACTION_RESUME: перед `startForeground()` - секундомер возобновлен
    - ACTION_APP_BACKGROUND: перед `startForeground()` - приложение в фоне с активным секундомером
  - ACTION_ALWAYS_ON НЕ запускает state listener - показывает статическое уведомление `buildAlwaysOnNotification()`
  - ACTION_PAUSE: изменена логика освобождения wake locks
    - PARTIAL_WAKE_LOCK всегда освобождается (не нужен CPU на паузе)
    - SCREEN_DIM_WAKE_LOCK сохраняется если useScreenDim=true (режим ALWAYS + dimBrightness=OFF)
    - Это предотвращает выключение экрана на паузе в режиме ALWAYS
  - State listener теперь создается только когда секундомер активен (время > 0 или работает)
  - Метод `startStateListener()` безопасен для повторных вызовов (отменяет предыдущий job)

## [0.9.5] - 2025-11-18

### Исправлено
- **Отображение миллисекунд на основном таймере**
  - Миллисекунды теперь показываются только при паузе
  - Во время работы секундомера миллисекунды скрыты для лучшей читаемости
- **Обновление времени при паузе**
  - Время теперь обновляется перед паузой для точного отображения миллисекунд
  - Исправлена проблема, когда время круга могло показывать больше, чем основное время
- **Уведомления при остановке секундомера**
  - Уведомление теперь корректно удаляется при нажатии кнопки "Стоп"
  - Исправлена проблема с повторным появлением уведомления после остановки
  - Все фоновые задачи теперь корректно отменяются перед удалением уведомления
- **Переключатель затухания яркости**
  - Исправлена логика работы переключателя для корректного затухания экрана во всех режимах
  - При включенном (иконка заполнена): яркость фиксирована на 10%, экран не затухает, не выключается
  - При выключенном (иконка контурная): яркость затухает естественно через системный таймаут, но экран не выключается
  - FLAG_KEEP_SCREEN_ON предотвращает затухание - используется только с dimBrightness = true
  - С dimBrightness = false используется SCREEN_DIM_WAKE_LOCK через сервис (разрешает затухание)
  - В режиме ALWAYS с dimBrightness = false: сервис работает даже при остановленном секундомере
  - В режиме WHILE_RUNNING: поведение остается как было - экран выключается при остановке
- **Уведомление при запуске приложения**
  - Исправлено появление уведомления "Paused" при запуске приложения с остановленным секундомером
  - Уведомление теперь появляется только когда есть активность (время > 0 или секундомер работает)
  - Различие между состояниями "стоп" (время = 0) и "пауза" (время > 0, не работает)
- **Race condition при остановке секундомера**
  - Исправлена проблема с повторным появлением уведомления при сворачивании приложения сразу после остановки
  - Состояние теперь сбрасывается синхронно, сохранение сессии происходит асинхронно
  - Корректная обработка сценария: работает → пауза → стоп → сворачивание приложения
- **Обновление wake locks при переключении настроек во время работы**
  - Переключатели теперь мгновенно влияют на поведение экрана даже когда секундомер работает
  - При изменении режима экрана (OFF/WHILE_RUNNING/ALWAYS) wake lock обновляется в реальном времени
  - При включении/выключении затухания яркости wake lock переключается между PARTIAL и SCREEN_DIM
  - Добавлен ACTION_UPDATE_WAKE_LOCK для обновления wake lock без перезапуска сервиса

### Технические детали
- **StopwatchScreen.kt**: изменена логика показа миллисекунд - `includeMillis = !isRunning`
- **StopwatchViewModel.kt**:
  - Добавлено обновление времени перед паузой в методах `pause()`, `addLapAndPause()` и обработчике команд
  - Убран вызов `pauseService()` при восстановлении состояния паузы
  - Исправлен race condition в методах `reset()` и обработчике `StopwatchCommand.Stop`
  - Состояние теперь сбрасывается синхронно перед асинхронным сохранением сессии
  - Метод `saveSession()` теперь принимает параметры вместо чтения из StopwatchState
  - Добавлен метод `shouldUseScreenDimWakeLock()` для определения когда использовать SCREEN_DIM_WAKE_LOCK
  - Логика передачи wake lock флага в сервис теперь учитывает screenOnMode и dimBrightness
  - Добавлен метод `updateServiceWakeLock()` для обновления wake lock при изменении настроек
  - Методы `cycleScreenOnMode()` и `toggleDimBrightness()` вызывают `updateServiceWakeLock()` если секундомер активен
- **StopwatchService.kt**:
  - Улучшена логика остановки сервиса - jobs отменяются с установкой в null перед вызовом `stopForeground()`
  - Добавлены проверки состояния в обработчиках ACTION_APP_FOREGROUND/BACKGROUND
  - ACTION_APP_BACKGROUND теперь вызывает `startForeground()` для корректного показа уведомления
  - Добавлен новый ACTION_ALWAYS_ON для режима ALWAYS с dimBrightness = false
  - Метод `buildAlwaysOnNotification()` создает минимальное уведомление "Screen stays on"
  - Добавлен ACTION_UPDATE_WAKE_LOCK для обновления типа wake lock без перезапуска сервиса
  - Обработчик освобождает текущий wake lock и захватывает новый на основе EXTRA_USE_SCREEN_DIM
- **MainActivity.kt**:
  - Условное использование FLAG_KEEP_SCREEN_ON в зависимости от dimBrightness
  - dimBrightness = true: FLAG_KEEP_SCREEN_ON + фиксированная яркость 0.1f
  - dimBrightness = false: без FLAG_KEEP_SCREEN_ON, полагается на SCREEN_DIM_WAKE_LOCK сервиса
  - FLAG_KEEP_SCREEN_ON действительно предотвращает затухание - нельзя использовать с dimBrightness = false
  - Добавлены проверки состояния секундомера перед отправкой ACTION_APP_FOREGROUND/BACKGROUND
- **StopwatchScreen.kt**:
  - Объединены обработчики screen mode и brightness в один колбэк
  - Передача dimBrightness в MainActivity для корректного управления wake locks
  - Добавлен LaunchedEffect для запуска сервиса в режиме ALWAYS с dimBrightness = false
  - Сервис запускается даже когда секундомер остановлен для удержания SCREEN_DIM_WAKE_LOCK

## Формат версий

- **MAJOR.MINOR.PATCH** (X.Y.Z)
  - **MAJOR** (X) - несовместимые изменения API
  - **MINOR** (Y) - новая функциональность (обратно совместимая)
  - **PATCH** (Z) - исправление ошибок (обратно совместимое)

- Версии **0.x.x** - pre-release (до публикации в Google Play)
- Версия **1.0.0** - первый публичный релиз в Google Play
